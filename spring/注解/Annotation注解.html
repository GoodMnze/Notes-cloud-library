Annotation注解
一、什么是注解
（1）是java5.0中新增的语法，它的行为十分类似public、final这样的修饰符
（2）它提供了一种安全的类似注释的机制，用来将任何的信息或元数据（metadata）与程序元素（类、方法、成员变量等）进行关联。
（3）更通俗的意思是为程序的元素（类、方法、成员变量）加上更直观更明了的说明（标记），这些说明信息是与程序的业务逻辑无关，并且是供指定的工具或框架使用的。
1、注解用于何处
    Annontation像一种修饰符一样，应用于元素对象（ElementType），如包、类型、构造方法、方法、成员变量、参数及本地变量的声明语句中。
2、注解类型
（1）annotation类型：
                        annotation类型定义了annotation的名字、类型、成员默认值
                        一个annotation类型是一个特殊的java接口
（2）annotation成员（是一个特殊的方法）
                        annotation的成员在annotation类型中以无参数的方法的形式被声明。
                        可以有默认值（使用default关键字）
（3）标记注解（marker annotation）类型：没有任何成员
【特点】
1、注解  不会  影响源代码
2、注解作用于程序元素
3、注解用于程序元素声明阶段
4、注解本身也有一个类型Annotation，定义方式与接口颇为相似

3、基本语法
public @interface 注解名称{
    public 类型 成员1()   [default  默认值];
    public 类型 成员2()   [default  默认值];
    ……
}
4、规则说明
（1）成员只有使用public或者默认的修饰符来修饰
（2）默认成员是value()
（3）每个成员都可以设置默认值，使用关键字default
（4）成员不能有参数，但是必须要有类型
（5）成员类型可以有八大基础数据类型、String、Class类型，enum枚举类型、Annotation类型，和以上所有类型的数组

//注解SelfAnnotation，可以作用于方法和属性上
//@Target({ElementType.METHOD,ElementType.FIELD})
@Target({METHOD,FIELD}) //上一句的简写，需要在导包处写上static
//运行期间依旧保留，可通过反射读取该注解
@Retention(RetentionPolicy.RUNTIME)
public @interface SelfAnnotation {
    //注解的成员
    public String value() default "abc";
    String[] names() default "六脉神剑";
    MyMark remark() default MyMark.YES;
}



5、注解的注解——元注解
1、@Target
    用于指定可以修饰的Java元素，不指定可以修饰的全部的Java元素（ElementType）
    Target类型有唯一的value成员，这个成员的类型是枚举ElementType[]类型的
    
public enum ElementType{
    TYPE，//类，接口（包括注解类型）或enum声明
    FIELD,//修饰属性（包括 enum 实例）
    METHOD，//修饰方法
    PARAMETER,//修饰参数
    CONSTRUCTOR,//修饰构造方法
    PACKAGE,//修饰包
    LOCAL_VAEIABLE;//本地变量
    ANNOTATION_TYPE;//修饰注解
}
 
使用@Target的写法：
//标准写法，表示该注解可应用于构造器、属性、本地变量、方法、包、参数还有类
@Target(value={CONSTRUCTOR, FIELD, LOCAL_VARIABLE, METHOD, PACKAGE, PARAMETER, TYPE})
//value是默认成员，所以可以省略直接写值
@Target({ElementType.METHOD,ElementType.FIELD})
//只有一个值，省略数组的{}
@Target(ElementType.METHOD)

 2、 @Retention
代表这个注解被保存的范围
public enum RetentionPolicy {
    /**
     * 源代码：即此注解只能保存在源代码中     * 当编译时，会被丢弃
     */
    SOURCE,
   /**
     * class文件：即此注解可以在class文件中保留     * 但会被jvm丢弃
     */
    CLASS,
   /**
     * 运行期：即此注解可以在运行时保留     * 可以通过反射获得
     */
    RUNTIME
}

二、Java内置的注解
A、@Override 
    告诉编译器定义了一个重写的方法，编译器会对语法进行检查 
B、@Deprecated
    表示已过期，不建议在程序中使用 
C、@SuppressWarnings 
    告诉编译器忽略警告
D、@WebServlet
    注解用于标注在一个继承了HttpServlet类之上，属于类级别的注解。
    用于标识当前Servlet的URl映射地址和其他相关信息

三、自定义注解

//注解SelfAnnotation，可以作用于方法和属性上
//@Target({ElementType.METHOD,ElementType.FIELD})
@Target({METHOD,FIELD}) //上一句的简写，需要在导包处写上static
//运行期间依旧保留，可通过反射读取该注解
@Retention(RetentionPolicy.RUNTIME)
public @interface SelfAnnotation {
    //注解的成员
    public String value() default "abc";
    String[] names() default "六脉神剑";
    MyMark remark() default MyMark.YES;
}
使用注解需要注意的点：
1）在注解名之前加@ 
2）成员是以属性名=属性值的形式出现 
3）如果没有指定属性名，则是以value属性赋值 
4）多个属性之间使用逗号（，）隔开 
5）属性值是不能动态改变的

/* 使用注解

public class UseAnnotation {
    //@SelfAnnotation(value = "古剑奇谭")
    @SelfAnnotation("古剑奇谭") //简写版本，默认给成员value赋值
    private Integer id;

    //只对names成员进行赋值，其他成员使用的是默认值
    @SelfAnnotation(names = "帝皇铠甲")
    public void info(){}

        //对所有成员都进行了赋值
    @SelfAnnotation
        (names={"凌","波","微","步"},
        value = "hyt",
        remark = MyMark.NO)
    public void show(){}
}


四、反射中获取注解
常用两个方法
1、判断Class对象的属性，是否包含注解
    public boolean isAnnotationPresent(Classs<? extends Annotation>)
2、获取修饰的注解
    public <A extends Annotation> getAnnotation(Class<A>)
 原则：修饰谁，谁负责
          Class c = SelfClass.class;
          // 修饰类
          if (c.isAnnotationPresent(SelfAnnotation.class)) {
              SelfAnnotation sa = (SelfAnnotation).getAnnotation(SelfAnnotation.class);
              System.out.println("获取成员desc：" + sa.desc());
              System.out.println("获取成员value：" + sa.value());
          }
          // 修饰方法
          Method m = c.getDeclaredMethod("show");
          if (m.isAnnotationPresent(SelfAnnotation.class)) {
              SelfAnnotation sa = (SelfAnnotation).getAnnotation(SelfAnnotation.class);
              System.out.println("获取成员desc：" + sa.desc());
              System.out.println("获取成员value：" + sa.value());
          } else {
              System.out.println("该方法不存在SelfAnnotation的注解信息");
          }
