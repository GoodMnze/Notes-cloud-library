IOC和DI
一、IOC
Ioc—Inversion of Control，即“控制反转”
一种思想，不是技术
将你设计好的对象交给容器控制，而不是传统的在你的对象内部直接控制
 控制权从应用程序转移到框架（如IOC容器），是框架共有特性。应用程序主动控制对象的实例化及依赖装配 

理解IOC容器问题关键：控制的哪些方面被反转了？
1、谁控制谁？为什么叫反转？                ------ IoC容器控制，而以前是应用程序控制，所以叫反转 
2、控制什么？                                         ------ 控制应用程序所需要的资源（对象、文件……）
3、为什么控制？                                      ------ 解耦组件之间的关系 
4、控制的哪些方面被反转了？                 ------ 程序的控制权发生了反转：从应用程序转移到了IoC容器。 

二、DI
DI—Dependency Injection，即“依赖注入”
用于构建Spring容器中对象之间的关系。用一个单独的对象（装配器）来装配对象之间的依赖关系 。
什么是注入？
    简单来说，就是“赋值”
理解DI问题关键
谁依赖于谁？                -------   应用程序依赖于IoC容器
为什么需要依赖？        -------   应用程序依赖于IoC容器装配类之间的关系
依赖什么东西？            -------   依赖了IoC容器的装配功能
谁注入于谁？                -------   IoC容器注入应用程序
注入什么东西？           -------   注入应用程序需要的资源（类之间的关系）


三、注入的方式
（一）构造器注入：调用构造方法来实现关系的建立
<!-- 构造器注入  ①index代表参数的位置，从0开始
               ②也可以type来指定参数的类型
               ③还可以通过参数名字来赋值  name="age" value="100"-->
<!--1、构造器注入方式案例-->
<bean id="clz" class="com.study.di.Clazz">
    <!--指定创建全参构造方法(Integer cid, String cname, String master)
        需要严格按照构造方法的顺序来赋值
    -->
    <constructor-arg index="0" value="9"/>
    <constructor-arg index="1" value="TR01601"/>
    <constructor-arg index="2" value="陈围"/>
</bean>

<bean id="clz2" class="com.study.di.Clazz">
    <!--根据参数类型赋值
        找构造方法中有一个Integer两个String的方法
        先遇见谁，就new谁
    -->
    <constructor-arg type="java.lang.Integer" value="101"/>
    <constructor-arg type="java.lang.String" value="TR1807"/>
    <constructor-arg type="java.lang.String" value="陈围2"/>
</bean>

<bean id="clz3" class="com.study.di.Clazz">
    <!--根据参数名字，推荐这种方式-->
    <constructor-arg name="cid" value="67"/>
    <constructor-arg name="cname" value="TR01702"/>
    <constructor-arg name="master" value="张佳玲"/>
</bean>
（二）setter方法注入：调用属性的setter方法来实现关系的建立
         name和属性无关，只和setter方法名有关
<bean id="stu" class="com.study.di.Student">
    <!--setter注入方式，和属性名字没有关系，之和set方法有关
        public void setId(Integer sid) {
        (1)去掉set关键字
        (2)首字母小写
    -->
    <property name="id" value="1"/>
    <!-- /*属性名：sAName*/
        public void setSAName(String name) -->
    <property name="SAName" value="大禹哥哥"/>
    <property name="sex" value="男"/>
    <!--birth 是一个复合数据类型 java.util.Date-->
    <property name="birth">
        <!--等同于 new java.util.Date()-->
        <bean class="java.util.Date"/>
    </property>
</bean>

四、注入其他类型
（一）一维数组和有序集合List
<!--注入一维数组 int[] lucyNumber-->
<property name="lucyNumber">
    <list>
        <value>7</value>
        <value>9</value>
    </list>
</property>
<!--注入List集合 List<String> friends-->
<property name="friends">
    <list>
        <value>叮当猫</value>
        <value>大雄</value>
        <value>胖虎</value>
    </list>
</property>


（二）无序集合
<!--注入Set集合 Set<String> movies-->
<property name="movies">
    <set>
        <value>僵尸先生</value>
        <value>逃学威龙</value>
    </set>
</property>


（三）Map集合
<!--注入Map集合 Properties-->
<property name="stars">
    <map>
        <entry key="男明星" value="古天乐"/>
        <entry key="女明星" value="钟楚红"/>
        <entry key="外国明星" value="孙艺珍"/>
    </map>
</property>


（四）Properties类型
属于HashTable的子类，键值对存储，等同于HashTable<String,String>
          <!-- stars:Properties -->
          <property name="stars">
              <props>
                   <prop key="男明星">古天乐</prop>
                   <prop key="女朋性">哈哈</prop>
              </props>
          </property>

（五）空值类型
<!-- 空值注入
<property name="xyz"><null/></property>-->

五、命名空间p
用来简化setter注入方式的
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
     xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
     xmlns:context="http://www.springframework.org/schema/context"
     xmlns:p="http://www.springframework.org/schema/p"
     xsi:schemaLocation="http://www.springframework.org/schema/beanshttp://www.springframework.org/schema/beans/spring-beans.xsd
http://www.springframework.org/schema/contexthttp://www.springframework.org/schema/context/spring-context-4.3.xsd">
     <!--
          P命名空间 简化setter写法。等同于
          <property name="name" value="xxx"/>
          <property name="sid">
              <value>9999</value>
          </property>
      -->
     <bean id="stu" class="com.study.iocanddi.Student" p:sid = "8" p:name="xxx"/>
</beans>

六、引入其他bean对象
<!--通过setter注入方式，给复合数据类型赋值-->
<!--java.util.Date d1 = new java.util.Date();-->
<bean class="java.util.Date" id="d1"/>
<!--java.sql.Date dd = new java.sql.Date(d1.getTime());-->
<bean class="java.sql.Date" id="dd">
    <constructor-arg index="0">
        <bean factory-bean="d1" factory-method="getTime"/>
    </constructor-arg>
</bean>

<!--p命名空间简写方式：本质上还是setter注入方式-->
<bean id="stu3" class="com.study.di.Student" p:id="3" p:SAName="兮兮" p:sex="男" p:birth-ref="d1" p:adultDate-ref="dd"/>




八、DI的优势
  • 动态替换Bean依赖对象，程序更灵活
  • 更好实践优先使用对象组合，而不是类继承
  • 采用对象组合，Bean的功能可能由几个依赖Bean的功能组合而成
  • 增加Bean可复用性
  • 降低Bean之间耦合
  • 代码结构更清晰
